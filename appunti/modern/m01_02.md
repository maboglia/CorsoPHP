## Scheda pratica 01.02 (Corso PHP)

# 1.2 Type Safety – Il PHP “Serio”

Obiettivo: imparare a scrivere PHP moderno **rigoroso**, leggibile e sicuro, riducendo errori logici e bug “silenziosi”.

---

# 1) Concetto chiave

PHP è nato come linguaggio permissivo (“fai quello che vuoi”), ma oggi è diventato un linguaggio **quasi da ingegneria software**, grazie a:

* Type hints (`int`, `string`, `bool`, classi…)
* Return type obbligatori
* Strict types
* Union types
* Intersection types
* Static analysis (PHPStan/Psalm)

---

# 2) Strict Types (fondamentale)

## Regola d’oro

Mettere SEMPRE in cima al file:

```php
<?php
declare(strict_types=1);
```

### Cosa cambia?

Senza strict types, PHP fa conversioni automatiche.

Esempio:

```php
function somma(int $a, int $b): int {
    return $a + $b;
}

echo somma("10", 5);
```

* **strict_types=0 (default)** → funziona (conversione automatica)
* **strict_types=1** → ERRORE (giusto!)

👉 Questo evita bug dove stringhe e numeri vengono mescolati senza accorgersene.

---

# 3) Type Hinting su parametri e return

## Standard moderno

```php
declare(strict_types=1);

function calcolaPrezzoFinale(float $prezzo, float $iva): float {
    return $prezzo + ($prezzo * $iva);
}
```

Regola: **ogni funzione seria deve avere tipo input e tipo output**.

---

# 4) Union Types (`int|float|string`)

Serve quando una variabile può essere più tipi.

Esempio tipico: input utente o JSON.

```php
declare(strict_types=1);

function normalizzaNumero(int|float $n): float {
    return (float)$n;
}

echo normalizzaNumero(10);
echo normalizzaNumero(10.5);
```

---

## Caso pratico: può arrivare stringa o numero

```php
declare(strict_types=1);

function parseId(int|string $id): int {
    return (int)$id;
}
```

👉 utile per API REST, query string, DB.

---

# 5) Tipo `mixed` (attenzione!)

`mixed` significa: “può essere qualunque cosa”.

```php
function stampa(mixed $dato): void {
    var_dump($dato);
}
```

⚠️ Non è “type safety”, è un ripiego quando non sai ancora cosa arriva.

Usalo solo se:

* input molto generico
* debug/log
* librerie di basso livello

---

# 6) Intersection Types (`A & B`)

Serve quando un oggetto deve implementare **più interfacce contemporaneamente**.

Esempio:

```php
declare(strict_types=1);

function contaElementi(Iterator&Countable $lista): int {
    return count($lista);
}
```

Significa:

* deve essere **Iterator**
* e deve essere anche **Countable**

👉 utile in librerie, framework, architetture avanzate.

---

# 7) Nullable types (`?string`)

Se un valore può essere `null`:

```php
declare(strict_types=1);

function trovaUtente(?string $email): string {
    if ($email === null) {
        return "email mancante";
    }
    return "utente trovato: $email";
}
```

Equivale a:

```php
string|null
```

---

# 8) Tipi sulle proprietà (Classi “serie”)

Oggi una classe PHP moderna deve avere proprietà tipizzate:

```php
declare(strict_types=1);

class Studente {
    public int $id;
    public string $nome;
    public float $media;
}
```

---

# 9) Return Types: mai senza

Errore comune da principianti:

```php
function faiQualcosa($x) {
    return $x * 2;
}
```

Versione corretta:

```php
function faiQualcosa(int $x): int {
    return $x * 2;
}
```

---

# 10) Perché farlo? (motivazione reale)

## Effetti concreti

* meno bug dovuti a conversioni automatiche
* meno errori in produzione
* codice più leggibile
* più facile da mantenere
* migliore supporto IDE (autocompletamento vero)
* più semplice scrivere test

📌 In pratica: gli errori vengono bloccati **prima di aprire il browser**.

# 11) Mini esercitazione da laboratorio

## Obiettivo

Scrivere una funzione sicura che calcoli lo sconto.

### Specifiche

* prezzo deve essere `float`
* sconto può essere `int|float`
* se sconto è > 100 → eccezione

---

### Soluzione esempio

```php
<?php
declare(strict_types=1);

function applicaSconto(float $prezzo, int|float $sconto): float {

    if ($sconto < 0 || $sconto > 100) {
        throw new InvalidArgumentException("Sconto non valido: $sconto");
    }

    return $prezzo - ($prezzo * ($sconto / 100));
}

echo applicaSconto(100, 20);
```

# 12) Checklist finale (da ricordare sempre)

✅ `declare(strict_types=1);` in ogni file
✅ tipi su parametri e return
✅ usare union types quando serve
✅ `mixed` solo se inevitabile
✅ proprietà delle classi tipizzate
✅ `?type` o `type|null` per i nullable
✅ eccezioni per input non valido
