# 1) Concetto chiave: “Syntax Sugar”

PHP 8 ha introdotto costrutti che permettono di:

* ridurre righe inutili
* evitare `if` annidati
* rendere il codice più espressivo
* ridurre errori “umani” (break mancanti, controlli null dimenticati…)

---

# 2) Match Expression (evoluzione di switch)

## Perché è migliore di switch?

`match` è:

* più compatto
* restituisce un valore direttamente
* non richiede `break`
* è più sicuro: se manca un caso può generare errore (se non c’è default)
* confronto **rigoroso** (`===`)

---

## Esempio base

```php
$status = match($code) {
    200 => 'Successo',
    404 => 'Non trovato',
    default => 'Errore ignoto',
};
```

---

## Confronto con switch (vecchio stile)

```php
switch($code) {
    case 200:
        $status = 'Successo';
        break;
    case 404:
        $status = 'Non trovato';
        break;
    default:
        $status = 'Errore ignoto';
}
```

👉 Più lungo, rischio di dimenticare `break`.

---

## Match con più valori sullo stesso caso

```php
$tipo = match($ext) {
    'jpg', 'png', 'gif' => 'immagine',
    'pdf' => 'documento',
    default => 'altro'
};
```

---

## Match senza default (modalità “rigorosa”)

```php
$messaggio = match($code) {
    200 => "OK",
    404 => "NOT FOUND"
};
```

Se arriva un valore diverso → errore (`UnhandledMatchError`)

👉 ottimo per logica controllata (stati, enum, ruoli).

# 3) Nullsafe Operator (`?->`)

## Problema classico

Quando abbiamo oggetti che possono essere null:

```php
if ($utente !== null) {
    if ($utente->getIndirizzo() !== null) {
        echo $utente->getIndirizzo()->getCitta();
    }
}
```

Codice brutto, lungo, difficile da leggere.

---

## Soluzione PHP 8: Nullsafe Operator

```php
echo $utente?->getIndirizzo()?->getCitta();
```

Significa:

* se `$utente` è `null`, restituisce `null`
* se `getIndirizzo()` restituisce `null`, restituisce `null`
* altrimenti restituisce la città

---

## Caso pratico: valore di fallback

```php
$citta = $utente?->getIndirizzo()?->getCitta() ?? "Città sconosciuta";
```

👉 perfetto per viste HTML o API JSON.

---

# 4) Named Arguments (parametri nominati)

## Problema classico

Funzioni con molti parametri sono difficili da leggere:

```php
creaUtente("Mario", "Rossi", 25, true, "Roma");
```

Chi legge non capisce cosa significhi ogni valore.

---

## Soluzione: Named Arguments

```php
creaUtente(
    nome: "Mario",
    cognome: "Rossi",
    eta: 25,
    attivo: true,
    citta: "Roma"
);
```

Ora il codice è autoesplicativo.

---

## Vantaggio enorme: ordine non obbligatorio

```php
creaUtente(
    cognome: "Rossi",
    nome: "Mario",
    citta: "Roma",
    attivo: true,
    eta: 25
);
```

---

## Caso pratico reale: funzioni con parametri opzionali

```php
setcookie(
    name: "token",
    value: "abc123",
    expires_or_options: time() + 3600,
    httponly: true,
    secure: true
);
```

---

## Attenzione (regola importante)

Non puoi mettere un argomento posizionale dopo uno nominato:

❌ Errato:

```php
creaUtente(nome: "Mario", "Rossi");
```

# 5) Mini esercitazione da laboratorio

## Obiettivo

Gestire una risposta HTTP e generare un messaggio utente.

---

### Codice esempio completo

```php
<?php
declare(strict_types=1);

function descriviRisposta(int $code): string {
    return match($code) {
        200 => "Operazione riuscita",
        201 => "Creato correttamente",
        400 => "Richiesta non valida",
        401 => "Non autorizzato",
        404 => "Risorsa non trovata",
        default => "Errore sconosciuto"
    };
}

class Profilo {
    public function __construct(private ?string $email) {}

    public function getEmail(): ?string {
        return $this->email;
    }
}

class Utente {
    public function __construct(private ?Profilo $profilo) {}

    public function getProfilo(): ?Profilo {
        return $this->profilo;
    }
}

$utente = new Utente(new Profilo(null));

$email = $utente?->getProfilo()?->getEmail() ?? "Email non presente";

echo descriviRisposta(404) . PHP_EOL;
echo $email . PHP_EOL;
```

# 6) Checklist finale (da ricordare)

✅ `match()` al posto di `switch` quando devi ottenere un valore
✅ `?->` per evitare catene di `if ($obj !== null)`
✅ Named Arguments per funzioni lunghe o con parametri opzionali
✅ `??` per valori di default in modo elegante
