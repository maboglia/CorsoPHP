# 2.3 Visibilità e Tipizzazione Rigida nelle Classi (PHP moderno)

Obiettivo: scrivere classi **robuste e professionali**, controllando accesso ai dati, prevenendo errori di tipo e proteggendo l’architettura con `final`.

---

# 1) Property Typing (tipizzare ogni proprietà)

## Concetto chiave

In PHP moderno, ogni proprietà deve avere un tipo esplicito.

📌 Regola pratica:

> Una proprietà senza tipo è una fonte di bug.

---

## Esempio base (corretto)

```php
<?php
declare(strict_types=1);

class Studente {

    public int $id;
    public string $nome;
    public string $email;
    public float $media;
}
```

---

## Perché è importante?

Senza tipizzazione:

```php
class Studente {
    public $media;
}
```

può succedere:

```php
$s = new Studente();
$s->media = "ventotto"; // nessun errore immediato...
```

Poi in produzione esplode.

Con tipo:

```php
$s->media = "ventotto"; // ERRORE immediato
```

✅ bug bloccato subito.

---

## Proprietà nullable (`?string`)

Se un valore può essere null:

```php
class Utente {
    public ?string $telefono = null;
}
```

---

## Proprietà con valore di default

```php
class Prodotto {
    public bool $attivo = true;
}
```

---

# 2) Visibilità: public / protected / private

## public

Accessibile ovunque.

```php
public string $nome;
```

📌 usalo per DTO semplici.

---

## private

Accessibile solo dentro la classe.

```php
private string $passwordHash;
```

📌 usalo per dati sensibili o logica interna.

---

## protected

Accessibile nella classe e nelle sottoclassi.

```php
protected PDO $pdo;
```

📌 utile in classi astratte o repository base.

---

# 3) Il Keyword `final` (protezione dell’architettura)

## Concetto chiave

`final` impedisce che una classe o un metodo venga esteso/sovrascritto.

---

## 3.1 Final Class

```php
final class AuthService {

    public function login(string $user, string $pass): bool {
        return true;
    }
}
```

Questo blocca:

```php
class FakeAuthService extends AuthService {} // ERRORE
```

---

## Quando serve una final class?

📌 quando vuoi evitare che qualcuno modifichi il comportamento previsto, ad esempio:

* servizi di sicurezza (login, token, permessi)
* DTO e Value Object
* classi core del framework/applicazione
* librerie interne aziendali

---

## 3.2 Final Method

Puoi impedire override solo su un metodo.

```php
class BaseController {

    final public function renderJson(array $data): void {
        echo json_encode($data);
    }
}
```

Ora le sottoclassi possono estendere la classe, ma non cambiare `renderJson()`.

---

## Vantaggio reale

✅ evita comportamenti imprevisti
✅ protegge parti critiche del sistema
✅ rende più stabile l’architettura

# 4) Variadic Arguments (`...$args`)

## Concetto

Permettono di accettare un numero indefinito di parametri.

---

## Esempio base

```php
function somma(int ...$numeri): int {
    return array_sum($numeri);
}

echo somma(10, 20, 30); // 60
```

Qui `$numeri` è un array automatico.

---

## Esempio: logger professionale

```php
class Logger {

    public function info(string $msg, mixed ...$dati): void {
        echo "[INFO] $msg\n";

        foreach ($dati as $d) {
            var_dump($d);
        }
    }
}
```

Uso:

```php
$log = new Logger();
$log->info("Utente creato", ["id" => 10, "nome" => "Mario"], "OK");
```

---

## Esempio pratico: costruire query dinamiche

```php
class QueryBuilder {

    public function where(string ...$conditions): string {
        return "WHERE " . implode(" AND ", $conditions);
    }
}

$q = new QueryBuilder();
echo $q->where("eta > 18", "citta = 'Roma'", "attivo = 1");
```

Output:

```sql
WHERE eta > 18 AND citta = 'Roma' AND attivo = 1
```

---

## Passare array a variadic (spread operator)

Puoi passare un array già pronto usando `...`

```php
$valori = [10, 20, 30];
echo somma(...$valori);
```

# 5) Mini esercitazione da laboratorio (completa)

## Obiettivo

Creare una classe DTO tipizzata + un service final + un metodo variadic.

---

### DTO tipizzato

```php
<?php
declare(strict_types=1);

readonly class StudenteDTO {

    public function __construct(
        public int $id,
        public string $nome,
        public string $email
    ) {}
}
```

---

### Service final con variadic

```php
final class StudenteService {

    public function stampaStudenti(StudenteDTO ...$studenti): void {
        foreach ($studenti as $s) {
            echo $s->id . " - " . $s->nome . " (" . $s->email . ")\n";
        }
    }
}
```

---

### Test

```php
$s1 = new StudenteDTO(1, "Mario", "mario@test.it");
$s2 = new StudenteDTO(2, "Anna", "anna@test.it");

$service = new StudenteService();
$service->stampaStudenti($s1, $s2);
```

# 6) Checklist finale (da ricordare)

✅ tipizzare sempre le proprietà
✅ usare `?tipo` se il valore può essere null
✅ usare `private` per proteggere dati interni
✅ usare `final class` per bloccare estensioni pericolose
✅ usare `final method` per proteggere metodi critici
✅ usare `...$args` per parametri variabili e pulizia del codice
