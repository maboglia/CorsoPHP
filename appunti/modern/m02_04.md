## Scheda pratica (Corso PHP)

# 2.4 Magic Methods: Il Potere Nascosto (PHP OOP)

Obiettivo: conoscere i **magic methods** (metodi speciali invocati automaticamente da PHP), capire quando sono utili e quando invece diventano una fonte di bug e confusione.

---

# 1) Cosa sono i Magic Methods?

I Magic Methods sono metodi con nome speciale (iniziano con `__`) che PHP chiama automaticamente in determinate situazioni.

📌 Regola importante:

> Sono potenti, ma vanno usati con disciplina.
> Abusarne porta a codice “magico” e difficile da debuggare.

---

# 2) `__construct()` e `__destruct()`

## `__construct()`

È il costruttore: viene eseguito quando crei un oggetto.

```php
class ConnessioneDB {

    public function __construct(private string $host) {
        echo "Connessione aperta verso $host\n";
    }
}

$db = new ConnessioneDB("localhost");
```

---

## `__destruct()`

Viene eseguito quando l’oggetto viene distrutto (fine script o garbage collector).

```php
class ConnessioneDB {

    public function __construct() {
        echo "Connessione aperta\n";
    }

    public function __destruct() {
        echo "Connessione chiusa\n";
    }
}
```

📌 Uso tipico:

* chiusura risorse
* flush log
* cleanup

⚠️ Attenzione:

* non è garantito il momento preciso (dipende dal ciclo di vita)
* in web app moderne spesso è inutile gestire manualmente risorse

---

# 3) `__get()` e `__set()` (accesso dinamico alle proprietà)

Questi metodi vengono chiamati quando provi a leggere o scrivere una proprietà **non accessibile** (inesistente o private/protected).

---

## Esempio pratico

```php
class Utente {

    private array $data = [];

    public function __get(string $name): mixed {
        return $this->data[$name] ?? null;
    }

    public function __set(string $name, mixed $value): void {
        $this->data[$name] = $value;
    }
}

$u = new Utente();
$u->nome = "Mario";   // chiama __set()
echo $u->nome;        // chiama __get()
```

---

## Perché è spesso meglio evitarli?

Perché:

* perdi il controllo sui tipi
* perdi autocompletamento IDE
* rendi il codice meno leggibile
* puoi introdurre bug invisibili (typo nei nomi)

Esempio pericoloso:

```php
$u->emali = "test@test.it"; // errore di battitura
```

Con `__set()` non esplode, ma salva un campo sbagliato → bug silenzioso.

---

## Alternativa migliore (consigliata)

Usare proprietà tipizzate + metodi chiari:

```php
class Utente {
    public function __construct(
        public string $nome,
        public string $email
    ) {}
}
```

Oppure getter/setter reali se necessario.

# 4) `__toString()` (oggetto trasformato in stringa)

PHP chiama `__toString()` quando l’oggetto viene usato come stringa.

---

## Esempio base

```php
class StudenteDTO {

    public function __construct(
        public int $id,
        public string $nome
    ) {}

    public function __toString(): string {
        return "$this->id - $this->nome";
    }
}

$s = new StudenteDTO(1, "Mario");
echo $s; // stampa: 1 - Mario
```

---

## Uso tipico

* log
* debug
* output CLI
* rappresentazione compatta dell’oggetto

📌 Ottimo per DTO, Entity, Value Object.

⚠️ Nota: deve sempre restituire una stringa (mai null).

---

# 5) `__invoke()` (oggetto eseguibile come funzione)

Permette di “chiamare” un oggetto come se fosse una funzione.

---

## Esempio semplice

```php
class Sommatore {

    public function __invoke(int $a, int $b): int {
        return $a + $b;
    }
}

$somma = new Sommatore();

echo $somma(10, 20); // 30
```

---

## Caso pratico: Validator riutilizzabile

```php
class EmailValidator {

    public function __invoke(string $email): bool {
        return filter_var($email, FILTER_VALIDATE_EMAIL) !== false;
    }
}

$validator = new EmailValidator();

if ($validator("mario@test.it")) {
    echo "Email valida\n";
}
```

---

## Caso pratico avanzato: Middleware / Pipeline

Molti framework usano `__invoke()` per rendere una classe “callable”:

```php
class AuthMiddleware {

    public function __invoke(array $request): array {
        $request["auth"] = true;
        return $request;
    }
}
```

# 6) Mini esercitazione da laboratorio

## Obiettivo

Creare un oggetto “stampabile” e “invocabile”.

---

### Codice completo

```php
<?php
declare(strict_types=1);

class ProdottoDTO {

    public function __construct(
        public int $id,
        public string $nome,
        public float $prezzo
    ) {}

    public function __toString(): string {
        return "Prodotto #$this->id: $this->nome (€$this->prezzo)";
    }
}

class ScontoApplicatore {

    public function __invoke(ProdottoDTO $p, float $percentuale): float {
        return $p->prezzo - ($p->prezzo * $percentuale / 100);
    }
}

$p = new ProdottoDTO(1, "Mouse", 20.00);

echo $p . PHP_EOL;

$applicaSconto = new ScontoApplicatore();
echo $applicaSconto($p, 10) . PHP_EOL;
```

Output atteso:

```
Prodotto #1: Mouse (€20)
18
```

# 7) Checklist finale (regole pratiche)

✅ `__construct()` sempre per inizializzare oggetti
⚠️ `__destruct()` solo per risorse particolari (log/cleanup)
⚠️ `__get()` e `__set()` da evitare nei progetti seri (rischio bug silenziosi)
✅ `__toString()` utilissimo per log e debug
✅ `__invoke()` utile per validator, middleware, funzioni riutilizzabili
