# 🗄️ Modulo 3: Gestione Dati e Database (The Modern Way)

L'obiettivo è interagire con il database senza mai compromettere la sicurezza e mantenendo il codice pulito attraverso l'astrazione.

## 3.1 PDO (PHP Data Objects): Lo Standard

PDO è l'interfaccia universale per connettersi a quasi ogni database (MySQL, PostgreSQL, SQLite).

* **Prepared Statements:** Perché non useremo mai più variabili direttamente nelle stringhe SQL (Prevenzione totale della SQL Injection).
* **Fetch Modes:** Trasformare i risultati del DB direttamente in Oggetti o Array associativi in modo automatico.
* **Gestione Errori:** Configurare PDO per lanciare Eccezioni (`PDOException`) invece di semplici warning silenziosi.

## 3.2 Transazioni e Integrità

* **ACID Principles:** Cosa succede se un trasferimento bancario si interrompe a metà?
* **BeginTransaction, Commit, Rollback:** Gestire operazioni atomiche dove "o tutto avviene con successo, o nulla viene salvato".

## 3.3 Introduzione all'ORM (Object-Relational Mapping)

Qui facciamo il salto verso i framework moderni (come Laravel o Symfony).

* **Il concetto di ORM:** Mappare una tabella del database a una Classe PHP e una riga della tabella a un Oggetto.
* **Active Record vs Data Mapper:** * *Eloquent (Laravel):* Semplicità e velocità.
* *Doctrine (Symfony):* Potenza e separazione delle responsabilità.


* **Query Builder:** Scrivere query complesse usando metodi PHP invece di stringhe SQL grezze.

## 3.4 Migrations: Il Version Control del Database

Dimentica l'export manuale di file `.sql` da phpMyAdmin.

* Cosa sono le Migrations: Scrivere la struttura del DB (tabelle, colonne, indici) in codice PHP.
* Vantaggi: Condividere la struttura del database con il team tramite Git e fare il "rollback" a una versione precedente in un secondo.

## 3.5 Performance: Indici e Caching

* **Database Indexing:** Quando e dove aggiungere indici per velocizzare le ricerche.
* **Introduzione a Redis:** Usare un database in-memory per salvare i risultati delle query più pesanti e abbattere i tempi di caricamento.

## 📝 Esercitazione Pratica del Modulo 3

**Il "Smart Inventory Manager":**
Creeremo un sistema di gestione magazzino che:

1. Utilizza **Migrations** per definire tabelle `products`, `categories` e `movements`.
2. Implementa una classe `DatabaseConnector` basata su **PDO** (Singleton pattern).
3. Gestisce il carico di un nuovo prodotto e l'aggiornamento delle scorte all'interno di una **Transazione SQL**.
4. Include una funzione di ricerca sicura che filtra i prodotti per nome e categoria usando **Prepared Statements**.
