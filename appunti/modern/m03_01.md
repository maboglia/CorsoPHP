# 3.1 PDO (PHP Data Objects): lo Standard per Database

Obiettivo: imparare a usare PDO in modo professionale per collegarsi al database, eseguire query sicure con **Prepared Statements**, gestire risultati con **Fetch Modes**, e configurare la gestione errori con **PDOException**.

---

# 1) Cos’è PDO?

PDO (PHP Data Objects) è lo standard moderno per lavorare con database in PHP.

Supporta:

* MySQL / MariaDB
* PostgreSQL
* SQLite
* SQL Server (driver permettendo)

📌 Vantaggio: impari PDO una volta e puoi cambiare DB con poche modifiche.

---

# 2) Connessione al Database con PDO

## Esempio MySQL/MariaDB

```php
<?php
declare(strict_types=1);

$dsn = "mysql:host=localhost;dbname=scuola;charset=utf8mb4";
$user = "root";
$pass = "root";

$pdo = new PDO($dsn, $user, $pass);
```

# 3) Gestione Errori: PDO deve lanciare eccezioni

Per default PDO può generare warning silenziosi.

📌 In un progetto serio, vogliamo eccezioni.

```php
$pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
```

Versione completa consigliata:

```php
<?php
declare(strict_types=1);

$dsn = "mysql:host=localhost;dbname=scuola;charset=utf8mb4";
$user = "root";
$pass = "root";

$pdo = new PDO($dsn, $user, $pass, [
    PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION
]);
```

Ora ogni errore SQL genera:

* `PDOException`

---

## Esempio try/catch

```php
try {
    $stmt = $pdo->query("SELECT * FROM studenti");
    $rows = $stmt->fetchAll(PDO::FETCH_ASSOC);
    print_r($rows);

} catch (PDOException $e) {
    echo "Errore DB: " . $e->getMessage();
}
```

# 4) Prepared Statements (anti SQL Injection)

## Problema gravissimo (da evitare)

Mai scrivere SQL così:

```php
$id = $_GET["id"];
$sql = "SELECT * FROM studenti WHERE id = $id";
```

❌ SQL Injection
❌ query instabile
❌ input utente non controllato

---

## Soluzione: Prepared Statements

```php
$id = 10;

$stmt = $pdo->prepare("SELECT * FROM studenti WHERE id = ?");
$stmt->execute([$id]);

$row = $stmt->fetch(PDO::FETCH_ASSOC);
print_r($row);
```

Qui il valore viene passato separatamente e PDO lo gestisce in sicurezza.

---

## Variante: placeholder nominati (consigliata)

```php
$stmt = $pdo->prepare("SELECT * FROM studenti WHERE id = :id");
$stmt->execute(["id" => 10]);

$row = $stmt->fetch(PDO::FETCH_ASSOC);
```

📌 Più leggibile e meno errori nei parametri.

---

## Inserimento sicuro (INSERT)

```php
$stmt = $pdo->prepare("
    INSERT INTO studenti(nome, email)
    VALUES(:nome, :email)
");

$stmt->execute([
    "nome" => "Mario Rossi",
    "email" => "mario@test.it"
]);

echo "Inserito con ID: " . $pdo->lastInsertId();
```

---

## Aggiornamento sicuro (UPDATE)

```php
$stmt = $pdo->prepare("
    UPDATE studenti
    SET email = :email
    WHERE id = :id
");

$stmt->execute([
    "email" => "nuova@test.it",
    "id" => 1
]);
```

---

## Cancellazione sicura (DELETE)

```php
$stmt = $pdo->prepare("DELETE FROM studenti WHERE id = :id");
$stmt->execute(["id" => 5]);
```

# 5) Fetch Modes (come ricevere i dati dal DB)

Quando fai una SELECT, PDO può restituire dati in modi diversi.

---

## 5.1 `PDO::FETCH_ASSOC` (array associativo)

```php
$stmt = $pdo->query("SELECT * FROM studenti");
$rows = $stmt->fetchAll(PDO::FETCH_ASSOC);

print_r($rows);
```

Output tipico:

```php
[
  ["id" => 1, "nome" => "Mario", "email" => "mario@test.it"],
  ["id" => 2, "nome" => "Anna", "email" => "anna@test.it"]
]
```

📌 È il modo più usato nelle API JSON.

---

## 5.2 `PDO::FETCH_NUM` (array numerico)

```php
$row = $stmt->fetch(PDO::FETCH_NUM);
```

Output:

```php
[0 => 1, 1 => "Mario", 2 => "mario@test.it"]
```

📌 Poco leggibile, ma più leggero.

---

## 5.3 `PDO::FETCH_OBJ` (oggetto generico)

```php
$stmt = $pdo->query("SELECT * FROM studenti");
$studente = $stmt->fetch(PDO::FETCH_OBJ);

echo $studente->nome;
```

📌 utile per prototipi rapidi.

---

## 5.4 `PDO::FETCH_CLASS` (mappare su una classe)

Puoi far costruire a PDO direttamente oggetti.

### DTO esempio

```php
class StudenteDTO {
    public int $id;
    public string $nome;
    public string $email;
}
```

Fetch diretto:

```php
$stmt = $pdo->query("SELECT id, nome, email FROM studenti");
$stmt->setFetchMode(PDO::FETCH_CLASS, StudenteDTO::class);

$lista = $stmt->fetchAll();
```

Ora `$lista` contiene oggetti `StudenteDTO`.

# 6) Esempio completo: Repository Pattern con PDO

## StudenteRepositoryPDO (professionale)

```php
<?php
declare(strict_types=1);

class StudenteRepositoryPDO {

    public function __construct(private PDO $pdo) {}

    public function findById(int $id): ?array {

        $stmt = $this->pdo->prepare("SELECT * FROM studenti WHERE id = :id");
        $stmt->execute(["id" => $id]);

        $row = $stmt->fetch(PDO::FETCH_ASSOC);

        return $row ?: null;
    }

    public function findAll(): array {
        $stmt = $this->pdo->query("SELECT * FROM studenti");
        return $stmt->fetchAll(PDO::FETCH_ASSOC);
    }
}
```

# 7) Mini esercitazione da laboratorio

## Obiettivo

Scrivere un file `test_pdo.php` che:

* si connette al DB
* legge tutti gli studenti
* stampa nome + email
* gestisce errori con try/catch

---

### Soluzione esempio

```php
<?php
declare(strict_types=1);

$dsn = "mysql:host=localhost;dbname=scuola;charset=utf8mb4";
$user = "root";
$pass = "root";

try {

    $pdo = new PDO($dsn, $user, $pass, [
        PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION
    ]);

    $stmt = $pdo->query("SELECT nome, email FROM studenti");

    $rows = $stmt->fetchAll(PDO::FETCH_ASSOC);

    foreach ($rows as $r) {
        echo $r["nome"] . " - " . $r["email"] . "\n";
    }

} catch (PDOException $e) {
    echo "Errore DB: " . $e->getMessage() . "\n";
}
```

# 8) Regole d’oro PDO (da fissare)

✅ sempre `PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION`
✅ mai concatenare input utente dentro SQL
✅ sempre prepared statements
✅ usare placeholder nominati per chiarezza
✅ scegliere il fetch mode in base allo scenario
✅ usare repository per separare SQL dalla logica applicativa
